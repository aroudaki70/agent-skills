# Design Sprint Nudges

A facilitation prompt library for each phase of the design sprint. Use these to guide the process adaptively.

## Phase 1: Map & Target Nudges

### Setting Context
- "What's the 2-year goal we're working toward?"
- "What does success look like in 2 years?"
- "What would we do if we had 10x the current resources?"

### Sprint Questions
- "What could make this fail?" (capture as sprint questions)
- "What must we learn to move forward?"
- "What's the riskiest assumption we're making?"

### Expert Q&A
- "Who has context we need? What questions do we have?"
- "What do we know for sure? What's still unclear?"
- "Who should we talk to before we go further?"

### Map Building (Design Track)
- "Walk me through the user journey from awareness to outcome."
- "Where are the friction points in this journey?"
- "What happens at each touchpoint?"

### Map Building (Dev Track)
- "What systems are involved in this flow?"
- "Where does data come from and go to?"
- "What are the integration points and dependencies?"

### Target Selection
- "Which moment has the most uncertainty and highest impact?"
- "Where do the HMW opportunities cluster?"
- "What would we learn most from testing here?"

## Phase 2: Sketch Nudges

### Inspiration Scan
- "What's working for competitors in this space?"
- "What adjacent industries solve similar problems?"
- "What analogous solutions exist in different domains?"

### Diverge Prompts
- "Go broad first - generate as many ideas as possible."
- "Crazy 8s: 8 variations in 8 minutes."
- "No bad ideas in sketch mode - capture everything."

### Refine Prompts
- "Which ideas have the most potential?"
- "What makes this solution compelling?"
- "What would we build if we had only 1 day?"

## Phase 3: Decide Nudges

### Sprint Participant Lenses

#### The User (Desirability)
- "Would users actually want this?"
- "Does this solve their real problem?"
- "Would they choose this over alternatives?"

#### The Builder (Feasibility)
- "Can we actually build this?"
- "What's the technical complexity?"
- "What technical risks exist?"

#### The Strategist (Viability)
- "Does this make business sense?"
- "What's the competitive advantage?"
- "Is this sustainable long-term?"

#### The Skeptic (Assumptions)
- "What could go wrong?"
- "What are we assuming that might not be true?"
- "What would make this fail?"

### Rumble vs Combine
- "Do we have two strong directions worth testing?"
- "Should we compare them side by side?"
- "Or should we combine the best elements?"

### Storyboard
- "What's the first step in this flow?"
- "Walk me through the experience step by step."
- "What happens at each screen/moment?"

## Phase 4: Prototype Nudges

### Goldilocks Quality
- "How real does this need to feel?"
- "What's the minimum to test our hypothesis?"
- "Don't overbuild - test the core assumption."

### Test Scenarios
- "What will users actually do with this?"
- "What specific behavior are we validating?"
- "Map each scenario to a sprint question."

## Phase 5: Validate Nudges

### Scoring
- "Did we validate, partially validate, or invalidate this?"
- "What's the evidence?"
- "Any surprises or unexpected findings?"

### Decision Framework
- **Iterate**: "We need to refine and test again."
- **Pivot**: "We learned something - new direction needed."
- **Ship**: "Validated enough to move to specs."
- **Kill**: "Not worth pursuing - stop here."

### Handoff
- "Ready to move to `spec-driven-development`?"
- "What do we carry forward from this sprint?"
- "What would we do differently next time?"

---

## Adaptive Depth Selection

### Quick Sprint (30-45 min)
Use: Focus on one key assumption, directional decision
Nudges: Skip inspiration scan, single concept, limited validation

### Standard Sprint (2-4 hours)
Use: Full 5-phase for most product work
Nudges: All phases, multiple concepts, user validation

### Deep Sprint (full day+)
Use: Major bets, multiple stakeholders, high uncertainty
Nudges: Extended phases, stakeholder reviews, multiple validation rounds

## Quick Reference

| Phase | Key Question | Outcome |
|-------|--------------|---------|
| Map & Target | Where should we focus? | Target moment + sprint questions |
| Sketch | What are our options? | Concept narratives |
| Decide | Which direction is best? | Storyboard |
| Prototype | What should we test? | Testable artifact |
| Validate | What did we learn? | iterate/pivot/ship/kill decision |
